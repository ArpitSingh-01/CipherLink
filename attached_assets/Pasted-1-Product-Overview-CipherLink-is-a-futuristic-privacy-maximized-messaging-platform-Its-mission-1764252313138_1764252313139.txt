2. Core Principles
2.1 Zero Identity

CipherLink does not use:

Emails

Phone numbers

Passwords

Unique usernames

OAuth logins

Identity = A cryptographic key pair generated locally.

2.2 Local-Only Username

User chooses a display name, but:

It is stored only in IndexedDB

Never uploaded

Never synced

Never verified

Server remains ignorant of usernames.

2.3 Private Friend Connections (No Search)

Users cannot search each other.

Instead, they connect via:

✔ One-time 8-character friend codes
✔ Codes auto-expire in 6 hours
✔ Codes become invalid once used
✔ Prevents scraping, discovery, enumeration

2.4 Self-Destructing Messages

Every message has a TTL:

10 seconds

30 seconds

5 minutes

1 hour

Expired messages automatically delete via Supabase scheduled cleanup.

2.5 Metadata Minimization (Supabase Log Retention Benefit)

Supabase free-tier auto-deletes logs every 24 hours.

This means:

IP logs

Lat/long

Network provider info

Request history

are all wiped without you needing to do anything.

This is a huge privacy benefit.

2.6 Multi-Device Support via Recovery Phrase

12-word BIP39 recovery phrase recreates:

Private key

Public key

Identity

No server-side backup exists.

3. User Flows
3.1 Onboarding
NEW USER FLOW

User opens site

App generates:

32-byte private key

Public key

12-word BIP39 phrase

User is shown recovery phrase and must store it offline

Keys stored in IndexedDB

Public key registered in Supabase users table

RESTORE FLOW

User selects “Restore Identity”

Enters 12-word phrase

Keys regenerated locally

Public key re-uploaded (if needed for device registration)

Server never sees or stores the recovery phrase.

3.2 Adding Friends (NO SEARCH)

User taps Add Friend

User generates an 8-character code (random, single-use)

Supabase stores:

code
identity_public_key
expires_at
used: false


Other user enters the code

Supabase links the two users

Code is auto-deleted

3.3 Messaging

User types a message

App encrypts using AES-256-GCM + X25519 key exchange

Uploads ciphertext with TTL

Real-time subscription notifies receiver

Receiver decrypts using private key

Supabase deletes expired messages automatically

3.4 Blocking Users

User taps “Block User”.

System performs:

Add blocked public key to blocklist table

Messages from blocked users not delivered

Real-time filters exclude blocked IDs

Friend relationship is frozen (cannot message)

User can unblock anytime.
